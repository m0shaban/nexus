## ملف ‎README‎ المحدث (النسخة الكاملة 2.0)

> الشعار
> **«حوِّل الفوضى إلى فهم، والفهم إلى تأثير.»**[^1]

---

### **المقدمة والمهمة**

نقطة الالتقاء – Nexus هو «مُركِّب أنظمة» (System Synthesizer) صُمِّم ليكون امتداداً حقيقياً لعقل «المستكشف الفضولي» (INTP): يلتقط كل شذرات المعلومات، يربطها شبكياً، ثم يدفعها إلى أرض الواقع عبر مشاريع وخطوات تنفيذية[^1][^3]. الهدف: سدّ الفجوة بين عالم الأفكار اللامحدود والتأثير العملي الملموس.

---

## الفلسفة الأساسية (The “Why”)

* الكفاءة فوق كل شيء: إدخال فكرة يجب أن يسهُل بقدر إرسال رسالة تيليجرام[^1].
* الروابط هي الجوهر: لا مجلدات معزولة، بل شبكة عصبونية تُبرز الأنماط الخفية[^1][^4].
* من الفهم إلى الفعل: الانتقال السلس من «ملاحظة» إلى «خطوات تنفيذية» لكسر شلل التحليل[^1][^3].

---

## البنية الوظيفية: الوحدات الأربع

| الوحدة | الغرض | المشكلة التي تُحلّ |
| :-- | :-- | :-- |
| **The Synapse** (المشبك) | التقاط كل المدخلات الخام وتحويلها إلى «ملاحظات حيّة» قابلة للربط[^1] | متلازمة الكائن اللامع، عبقرية الصومعة |
| **The Catalyst** (المُحفّز) | صهر الملاحظات الناضجة إلى مشاريع ومهام قابلة للتنفيذ[^1] | مقبرة المشاريع غير المكتملة، الشلل التحليلي |
| **The Oracle** (العرّافة) | صندوق رمل لمحاكاة المستقبل وتحليل المخاطر قبل الالتزام بالموارد[^1][^3] | سوء تقدير المخاطر، القرارات الانفعالية |
| **The Mirror** (المرآة) | لوحة قيادة للوعي الذاتي وتحليل السلوك والمزاج على المدى الطويل[^1][^4] | غياب التعاطف اللحظي، إهمال الحاضر |

### تفاصيل كل وحدة

**The Synapse**[^1]

- التقاط عبر بوت تيليجرام (نص، روابط، صور).
- تلخيص فوري + 3 أسئلة استفزازية عند الطلب.
- خريطة أفكار ثلاثية الأبعاد وروابط ذكية استباقية (رؤية مستقبلية).

**The Catalyst**[^1]

- «قمع من الفكرة إلى الخطوة الأولى» يولِّد ثلاث خطوات قابلة للتنفيذ خلال ‎48‎ ساعة.
- «سلسلة الإنجاز» تحفّز على الاستمرارية.
- مدرب-مساءلة يرسل تحليل الأسباب عند انقطاع السلسلة.

**The Oracle**[^3]

- صندوق رمل سيناريوهات مع أفضل/أسوأ/أغلب النتائج.
- «Pre-Mortem AI» يولِّد قائمة المخاطر المحتملة قبل البدء.
- رادار اتجاهات يقتنص «الإشارات الضعيفة» من موجزات المعرفة.

**The Mirror**[^4]

- يوميات موجهة بأسئلة شخصية ديناميكية.
- تحليل مشاعر وسلوك طويل الأجل وإشعارات نمو مبنية على البيانات.
- لوحة قيادة شخصية تجمع السلاسل، الروابط الذكية، والخطوات التالية الأكثر إلحاحاً.

## الهندسة المعمارية التقنية

* **الإطار:** Next.js (TypeScript) مع وضع App Router لدمج SSR, SSG, API routes في حزمة واحدة[^1][^3].
* **الواجهة:** React + Tailwind CSS + Shadcn/UI لتصميم سريع ونظيف[^1].
* **البيانات:** Supabase ‏(Postgres، Auth، Storage، Realtime subscriptions)[^1][^3].
* **النشر:** Vercel لتكامل Git ونشر فوري[^1].
* **الذكاء الاصطناعي:**
    * التحليل النصي: ‎nvidia/llama-3.1-nemotron-ultra-253b-v1 عبر ‎NVIDIA AI API‎ لما يتميز به من قدرات استدلال عميقة[^3].
    * سيناريوهات ومحاكاة: نفس النموذج مع إعدادات درجة حرية عالية.
* **ذراع الالتقاط السريع:** بوت تيليجرام متصل بـ Webhook Serverless[^1].
* **التطبيق كـ PWA:** يمكن تثبيته على الهاتف أو سطح المكتب ويعمل دون اتصال أساسي[^1].

## خارطة طريق التطوير المرحلية

### المرحلة 1 — The Synapse (MVP)

1. تهيئة مشروع Next.js مع ‎TypeScript‎ وTailwind[^1].
2. إعداد Supabase وتمكين RLS[^1].
3. إنشاء ‎/api/telegram-webhook‎ لحفظ الرسائل في جدول ‎notes‎[^1].
4. واجهة ويب تعرض الملاحظات آنياً باستخدام Realtime[^1].
5. Route ‎/api/analyze‎ يتصل بـ NVIDIA AI لتوليد ملخص وأسئلة[^3].
**Definition of Done:** إرسال ملاحظة من تيليجرام ➜ ظهورها فورياً ➜ تحليلها بزر واحد.

### المرحلة 2 — The Catalyst

* جداول ‎projects‎ و‎tasks‎.
* زر «حوّل إلى مشروع» + صفحة ‎/projects‎[^3].
* ‎/api/projects/plan‎ يولّد المهام الأولية بثلاث خطوات ملموسة[^3].
* سلسلة الإنجاز وCoach الإشعارات.


### المرحلة 3 — The Oracle

* جدول ‎scenarios‎.
* مكون «Scenario Sandbox» داخل صفحة المشروع.
* ‎/api/scenarios/pre-mortem‎ لإنتاج قائمة المخاطر[^3].


### المرحلة 4 — The Mirror

* جدول ‎journal_entries‎ وشاشة ‎/dashboard‎[^3].
* ‎/api/journal/reflect‎ لتحليل المزاج والسلوك.
* مخطط المشاعر وتوصيات النمو.

## قصص المستخدم (الرؤية الكاملة)

* كمستخدم، أرسل فكرة إلى تيليجرام وأراها فوراً في Nexus[^2].
* كمستخدم، أحوِّل ملاحظة ملهمة إلى مشروع بنقرة ويحصل على قائمة مهام مقترَحة[^3].
* كمستخدم، قبل أن أبدأ مشروعاً كبيراً، أطلب من الذكاء الاصطناعي كشف الأسباب المحتملة لفشله[^3].
* كمستخدم، أكتب يومياتي وأتلقى انعكاساً ذكياً يساعدني على فهم نفسي[^3].

## إعداد بيئة التطوير

1. `git clone …`
2. `npm install`
3. إنشاء ‎`.env.local`‎ ووضع القيم:
```
NEXT_PUBLIC_SUPABASE_URL=…
NEXT_PUBLIC_SUPABASE_ANON_KEY=…
SUPABASE_SERVICE_ROLE_KEY=…

TELEGRAM_BOT_TOKEN=…
TELEGRAM_WEBHOOK_SECRET=…

NVIDIA_API_KEY=…
NVIDIA_API_BASE_URL=https://integrate.api.nvidia.com/v1
```

4. `npm run dev` ثم افتح http://localhost:3000.
5. عيّن Webhook البوت إلى ‎https://your-deploy.vercel.app/api/telegram-webhook‎.
